##### 전략
````
N-Queen 문제는 크기가 N × N인 체스판 위에 퀸 N개를 서로 공격할 수 없게 놓는 문제이다.
N이 주어졌을 때, 퀸을 놓는 방법의 수를 구하는 프로그램을 작성하시오.

예를 들어 n = 4 이면, 4 * 4 = 16 개의 퀸을 놓을 수 있다.
그 중에서 서로 공격할 수 없게 놓는 경우를 찾아야 한다.

- 생각
1. 16개의 퀸을 놓을 때 첫번째 놓는 퀸의 모든 경우의 수를 찾고, 서로 공격 할 수 없게 놓아진 경우만 더해서 출력하는 방법?
-> 2차원 배열을 생성
-> ex) 첫번째 놓는 퀸을 놓는 경우의수 (0,0) (0,1) (0,2) ... (3,2) (3,3) 총 16가지
-> 16가지에 대한 반복문
-> 퀸을 놓을 수 있는지 여부에 대한 조건문
-> 하지만, 중복이 되는 경우의 수가 생길 것이고(반복제거에 대한 우려?), 또한 너무 많은 반복이 생긴다. 
-> 중복에 대한 탐색을 모두 실행 하지 않고, 가능한 후보군으로 가지를 치며 탐색하는 백트래킹을 활용해야한다.
-> 재귀함수를 이용하여 해를 찾아가는 도중 해가 될 것 같지 않은 경로가 있다면 더 이상 가지 않고 되돌아가는 방법!

2. 가능한 후보군을 위한 규칙을 찾는다.
-> 2차원 배열로 받지 않고, 1차원 배열로 받는다. 왜냐하면 같은 행과 같은열에는 퀸을 놓을 수 없다.
-> 1차원 배열에 각 인덱스(위치)는 '열'을 가리키고, 해당 인덱스의 값은 '행'을 의미한다. 
-> 4*4 체스판이라고 했을때, 서로 공격할 수 없게 놓는 하나의 경우는 (2,0)(0,1)(1,3)(3,2) 이다.
-> 따라서 {2,0,3,1}이라는 1차원 배열을 만들 수 있다.
-> 재귀와 조건에 대한 문제 
    1. 재귀호출을 어떻게 할 것인가
        -> 퀸을 놓을 수 있는 상황일때,
    2. 퀸을 놓을 수 있는지 여부를 어떻게 조건문으로 판별 할 것인가
        -> 1. 같은 행에 존재 할 경우
        -> 2. 같은 열에 존재 할 경우 (1차원 배열이기 때문에 이미 판별완료)
        -> 3. 대각선 상에 놓여 있는 경우 (열의 차와 행의 차가 같을 경우)
````
